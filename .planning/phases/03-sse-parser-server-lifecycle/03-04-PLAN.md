/**

 * Constructor-injected dependencies for testability (D-10, D-27).

class FOpenCodeServerLifecycle
{
public:
    /**
     * @param InProcessLauncher - Process launcher (mock in tests, real in production)
     * @param InHttpClient - HTTP client for health checks (mock in tests)
     * @param InOpenCodePath - Path to opencode binary
     */
    FOpenCodeServerLifecycle(
        TSharedPtr<IProcessLauncher> InProcessLauncher,
        TSharedPtr<IHttpClient> InHttpClient,
        const FString& InOpenCodePath);
    ~FOpenCodeServerLifecycle();

    /** Start detection/spawn sequence. Transitions from Disconnected. */
    void EnsureServerReady();

    /** Shutdown managed server. Transitions to Disconnected. */
    void Shutdown();

    /** Get current connection state */
    EOpenCodeConnectionState GetConnectionState() const;

    /** Get the base URL for the connected server (e.g., http://127.0.0.1:4096) */
    FString GetServerBaseUrl() const;

    /** Get next backoff delay in seconds */
    float GetNextBackoffSeconds() const;

    /** Record a reconnect failure (increments backoff) */
    void IncrementBackoff();

    /** Reset backoff to initial value (on successful connect) */
    void ResetBackoff();

    /** Transition to Reconnecting state */
    void MarkReconnecting();

    /** Transition to Connected state */
    void MarkConnected();

    /** Transition to Connecting state */
    void MarkConnecting();

    FOnConnectionStateChanged OnStateChanged;

**Implementation (OpenCodeServerLifecycle.cpp):**

1. **Constructor**: Store injected dependencies. Initialize `CurrentBackoffSeconds = SSEReconnectBackoffInitialSeconds`.

2. **EnsureServerReady()**: If not Disconnected, return. Set state to Detecting. Call DetectExistingServer(). If detected → set state to Connecting. If not → call TrySpawnServer(). If spawned → set state to Connecting. If failed → set state to Disconnected + fire error.

3. **DetectExistingServer()**: Read PID file. If PID file exists:
   - Check IsApplicationRunning(PID). If dead → delete PID file, return false.
   - If alive → check health via HttpClient->Get(BaseUrl/global/health). If healthy → set ActivePort, bIsManagedServer=true (we have PID file = we spawned it previously), return true.
   - If alive but unhealthy → kill process (TerminateProc), delete PID file, return false (D-09).
   - If no PID file → return false.

4. **TrySpawnServer()**: Resolve port. Loop MaxPortRetryCount times. For each port, call ProcessLauncher->LaunchProcess with `opencode serve --port N --hostname 127.0.0.1`. If handle valid → store handle, PID, port, write PID file, set bIsManagedServer=true, return true. If all fail → fire error ServerUnreachable, return false.

5. **ResolvePort()**: Check ConfiguredPort > 0 → return. Check env var OPENCODE_PORT → parse and return. Return DefaultOpenCodePort (4096).

6. **Shutdown()**: If bIsManagedServer and ManagedPID > 0: TerminateProc, delete PID file. Set state Disconnected. Clear ManagedHandle, ManagedPID, bIsManagedServer.

7. **Backoff methods**: GetNextBackoffSeconds returns CurrentBackoffSeconds. IncrementBackoff doubles CurrentBackoffSeconds (capped at max). ResetBackoff sets to initial. MarkReconnecting/MarkConnected/MarkConnecting transition state.

8. **PID file path**: `FPaths::ProjectSavedDir() / TEXT("UnrealClaude") / OpenCodePidFilename`.

IMPORTANT: The FProcHandle returned by FMockProcessLauncher is default-constructed (invalid on real UE). The lifecycle must track spawn success via a `bSpawnSucceeded` flag set from the mock's `bLaunchShouldSucceed`, not solely from `Handle.IsValid()`. In production, the real ProcessLauncher returns a valid handle from CreateProc. In the mock, we use `bLaunchShouldSucceed` to control behavior and the lifecycle checks it via the mock's LaunchCallCount + the bLaunchShouldSucceed state. Alternatively, the lifecycle can check if OutPID != 0 as a spawn success indicator.
