From 03-RESEARCH.md — OpenCode SSE Event Mapping:
```
server.connected       → SessionInit
message.part.updated (TextPart)     → TextContent   (part.content + delta)
message.part.updated (ToolPart)     → ToolUse       (part.callID, part.tool, part.state)
message.part.updated (ToolPart, state=completed) → ToolResult (part.output)
message.part.updated (StepStartPart/StepFinishPart) → AgentThinking
message.part.updated (ReasoningPart) → AgentThinking
message.updated       → AssistantMessage
permission.updated    → PermissionRequest
session.status        → StatusUpdate
session.error         → bIsError=true + EOpenCodeErrorType
```

SSE Wire Format:
```
data: {"type":"message.part.updated","properties":{...}}\n\n
```
- Lines starting with `data:` carry JSON payload
- Lines starting with `event:` carry event name (fallback; JSON `type` wins)
- Lines starting with `:` are comments — ignore
- Lines starting with `retry:` carry reconnect interval in ms
- Blank line `\n\n` terminates an event

**Header (OpenCodeSSEParser.h):**
```cpp
class FOpenCodeSSEParser
{
public:
    /** Delegate fires once per complete SSE event */
    FOnChatStreamEvent OnEvent;

    /** Feed raw bytes from HTTP stream delegate. Called from any thread. */
    void ProcessChunk(const FString& RawChunk);

    /** Reset buffer state (call on reconnect) */
    void Reset();

    /** Get last retry interval from SSE retry: field (0 if none) */
    uint32 GetLastRetryMs() const { return LastRetryMs; }

private:
    FString Buffer;
    FString CurrentEventName;
    FString CurrentData;
    uint32 LastRetryMs = 0;

    void FlushEvent();
    FChatStreamEvent ParseEventPayload(const FString& EventName, const FString& Data);
    EChatStreamEventType MapEventType(const FString& TypeStr, const TSharedPtr<FJsonObject>& Payload);
    FChatStreamEvent MapMessagePartUpdated(const TSharedPtr<FJsonObject>& Payload);
    FChatStreamEvent MapSessionStatus(const TSharedPtr<FJsonObject>& Payload);
    FChatStreamEvent MapSessionError(const TSharedPtr<FJsonObject>& Payload);
};
```

**Implementation (OpenCodeSSEParser.cpp):**
Follow the ProcessChunk algorithm from RESEARCH.md exactly:
1. Append RawChunk to Buffer
2. While Buffer contains `\n`: extract line, process field (`data:`, `event:`, `retry:`, `:`, or blank)
3. Blank line calls FlushEvent() which parses accumulated data fields

FlushEvent(): Parse CurrentData as JSON → extract `payload` if present → read `type` → call MapEventType → fire OnEvent delegate.

MapMessagePartUpdated(): Read `properties.part` from payload. Check part shape:
- Has `content`/`delta` (text fields) → TextContent
- Has `callID`/`tool` with state != "completed" → ToolUse
- Has `callID`/`tool` with state == "completed" → ToolResult
- Has `type: "step-start"` or `type: "step-finish"` → AgentThinking
- Has `type: "reasoning"` → AgentThinking

MapSessionError(): Classify error code to EOpenCodeErrorType based on status code in error payload.

IMPORTANT: Do NOT use AsyncTask/GameThread dispatch in the parser itself — that's the caller's responsibility (Phase 4's FOpenCodeRunner). The parser is a pure data transformer. Tests rely on synchronous delegate firing.
