**Create `Private/OpenCode/OpenCodeRunner.cpp`** — Full implementation. This is the composition task. Reference `ClaudeCodeRunner.cpp` for threading patterns. Key sections:

**Constructor** (per D-15, Pitfall 7):
- If `InHttpClient` is nullptr, create `MakeShared<FOpenCodeHttpClient>()` (production). Same for `InProcessLauncher` → create `MakeShared<FDefaultProcessLauncher>()` (or the production launcher).
- Get OpenCode binary path via `FChatBackendFactory::GetOpenCodePath()`.
- Create `Lifecycle = MakeUnique<FOpenCodeServerLifecycle>(ProcessLauncher, HttpClient, OpenCodePath)`.
- Bind `SSEParser.OnEvent` to `HandleSSEEvent()`.
- Bind `Lifecycle->OnStateChanged` to `OnConnectionStateChanged()` — do NOT call `EnsureServerReady()` in constructor (lazy init).
- Note: `FOnConnectionStateChanged` delegate takes a single `EOpenCodeConnectionState` param (the new state).

**Destructor** (per Pitfall 6 — strict order):
1. `HttpClient->AbortSSEStream()`
2. `StopTaskCounter.Increment()`
3. If Thread: `Thread->WaitForCompletion()`, then `delete Thread`
4. `Lifecycle->Shutdown()`

**Initialize()** — called from first `ExecuteAsync()`:
- Guard: if already initialized (Lifecycle state != Disconnected), return.
- Call `Lifecycle->EnsureServerReady()` — kicks off detect/spawn sequence.

**OnConnectionStateChanged(EOpenCodeConnectionState NewState)** (per D-16):
- `Connected`: Call `RegisterMcpBridge()` (D-08), then `CreateOrReuseSession()` (D-02), then `StartSSEStream()`.
- `Reconnecting`: Call `SSEParser.Reset()`. On next Connected transition, re-register MCP (D-09).
- `Disconnected`: Call `HttpClient->AbortSSEStream()`, clear `ActiveSessionId`, call `SSEParser.Reset()`.

**RegisterMcpBridge()** (per D-08, D-09, D-10, D-11, MCPI-01):
- Construct MCP URL: `FString::Printf(TEXT("http://localhost:%d"), UnrealClaudeConstants::MCPServer::DefaultPort)`
- POST body: `{"name":"unrealclaude","config":{"type":"remote","url":"<MCP_URL>","enabled":true}}`
- POST to `Lifecycle->GetServerBaseUrl() + TEXT("/mcp")`
- On failure (status != 200 && != 201): log warning with `UE_LOG(LogUnrealClaude, Warning, ...)`, continue (D-10 graceful degradation). Do NOT abort.

**CreateOrReuseSession()** (per D-02, D-03):
- If `ActiveSessionId` is non-empty, return (reuse per D-02).
- POST `{"title":"UnrealClaude"}` to `Lifecycle->GetServerBaseUrl() + TEXT("/session")`
- Parse response JSON for `"id"` field → store in `ActiveSessionId`.
- On failure: log warning, leave `ActiveSessionId` empty (will retry on next prompt per D-03).

**StartSSEStream()** (per D-18, Pitfall 2):
- Call `HttpClient->BeginSSEStream()` with URL `Lifecycle->GetServerBaseUrl() + TEXT("/global/event")` (NOT `/event` — per Pitfall 2).
- Stream callback: convert `void* Ptr, int64 Length` to `FString` (UTF-8 → TCHAR), feed to `SSEParser.ProcessChunk()`. Return `StopTaskCounter.GetValue() == 0`.
- Disconnect callback: Mark `Lifecycle->MarkReconnecting()`, then schedule reconnect via backoff.

**ExecuteAsync()** (per D-18, COMM-01):
- If `bIsExecuting`, return false (already running).
- Call `Initialize()` if not yet initialized (lazy init, per Pitfall 7).
- If `ActiveSessionId.IsEmpty()`, call `CreateOrReuseSession()`. If still empty, call `OnComplete` with error and return false.
- Store config as `FOpenCodeRequestConfig` (static_cast from `FChatRequestConfig` — same pattern as `FClaudeCodeRunner` uses for `FClaudeRequestConfig`). Copy delegates.
- Set `bIsExecuting = true`, reset `StopTaskCounter`.
- Build POST body via `BuildMessageBody()` and POST to `Lifecycle->GetServerBaseUrl() + TEXT("/session/") + ActiveSessionId + TEXT("/message")`.
- The POST is synchronous (IHttpClient::Post blocks). Once it returns, the response contains the final message. Parse it and call `ReportCompletion()`.
- NOTE: Using synchronous `/message` endpoint. SSE stream is already running and dispatches streaming events to UI in real-time. The POST provides the final completion signal.
- On success, set `bIsExecuting = false`, fire `OnCompleteDelegate` on game thread.
- On failure (non-200 status): if 401/403 → set `ActiveSessionId` empty (D-03 — session invalidated, will recreate next time). Fire `OnCompleteDelegate` with `bSuccess=false`.
- Do NOT spawn a `FRunnableThread` for `ExecuteAsync` — run the HTTP POST on a `FRunnableThread` to avoid blocking game thread. Create thread: `Thread = FRunnableThread::Create(this, TEXT("OpenCodeRunner"))`. The `Run()` method performs the POST.

**Run()** (FRunnable — worker thread):
- Check `StopTaskCounter`. If set, return 0.
- Build message body from `CurrentConfig`.
- POST to `/session/{ActiveSessionId}/message`.
- Parse response.
- Call `ReportCompletion()` via `AsyncTask(GameThread)`.
- Set `bIsExecuting = false`.
- Return 0.

**ExecuteSync()** (blocking variant):
- Same as ExecuteAsync but runs on calling thread. POST, parse, return result directly.
- Guard: if `ActiveSessionId.IsEmpty()`, try `CreateOrReuseSession()`.

**Cancel()** (per CHUI-03, Pitfall):
- If `ActiveSessionId` is non-empty: POST `{}` to `Lifecycle->GetServerBaseUrl() + TEXT("/session/") + ActiveSessionId + TEXT("/abort")` (NOT `/cancel` — per Research Pitfall 1).
- Call `HttpClient->AbortSSEStream()`.
- Increment `StopTaskCounter`.
- Set `bIsExecuting = false`.

**IsAvailable()** (per D-17):
- Return `Lifecycle && Lifecycle->GetConnectionState() == EOpenCodeConnectionState::Connected`.

**GetDisplayName()**: Return `TEXT("OpenCode")`.
**GetBackendType()**: Return `EChatBackendType::OpenCode`.

**CreateConfig()** (per D-14):
- Return `MakeUnique<FOpenCodeRequestConfig>()`.

**FormatPrompt()** (per D-04):
- Same Human:/Assistant: format as FClaudeCodeRunner. Iterate `History` pairs, append `"Human: " + Key + "\n\n" + "Assistant: " + Value + "\n\n"`. Return formatted string.

**BuildMessageBody()** (per COMM-06):
- Build JSON: `{"parts":[{"type":"text","text":"<escaped_prompt>"}],"model":"<ModelId>","agent":"<AgentType>"}`.
- Use `EscapeJsonString()` for the prompt text.

**HandleSSEEvent()** (per D-05, D-06):
- If `Event.Type == EChatStreamEventType::PermissionRequest`: extract permission ID from `Event.RawJson` (parse JSON → `properties` object → `id` string — per Pitfall 4), set `bPermissionDialogActive = true`, dispatch to game thread via `AsyncTask` to show modal dialog. After user responds, POST allow/deny to `/session/{ActiveSessionId}/permissions/{permId}` (per D-07). Then drain `PendingPermissionBuffer` and set `bPermissionDialogActive = false`.
- If `bPermissionDialogActive`: lock `PermissionBufferLock`, add event to `PendingPermissionBuffer` (per D-06).
- Otherwise: call `DispatchEventToUI(Event)`.

**DispatchEventToUI()** (game thread dispatch):
- Copy delegates to local vars.
- `AsyncTask(ENamedThreads::GameThread, ...)`: fire `OnStreamEvent` delegate, and if `TextContent` type, also fire `OnProgressDelegate`.
- If `Event.Type == EChatStreamEventType::Result`: this signals completion from the SSE stream.

**ShowPermissionDialog()** (per D-05, Research Open Question 3):
- Use `FMessageDialog::Open(EAppMsgType::YesNo, FText::FromString(PermissionDescription))`.
- Extract description from Event.Text or Event.RawJson.
- Return `true` if `EAppReturnType::Yes`.

**ExtractPermissionId()** (per Pitfall 4):
- Parse `RawJson` as JSON → get `"properties"` object → get `"id"` string.
- Return empty string on failure.

**EscapeJsonString()**:
- Escape backslash, double-quote, newline, tab, carriage return for JSON string embedding.

Include `UnrealClaudeModule.h` for `LogUnrealClaude` (per Phase 3 decision — do not re-declare).
